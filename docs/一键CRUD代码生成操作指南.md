# Gin-Vue-Admin：一键 CRUD（代码生成器）操作指南

本文说明在本框架中，如何通过**系统工具 → 代码生成器**从数据表生成前后端 CRUD 代码，并把**菜单 / API / 权限**串起来，使左侧菜单可点击进入管理页面。

> 说明：框架**不是**在任意业务菜单上点一下就生成 CRUD；入口固定为 **「系统工具」下的「代码生成器」**。生成后如需出现在侧边栏，需依赖「自动创建菜单」或手动配置菜单，并在角色里授权。

---

## 一、功能入口与相关菜单

| 菜单位置 | 路由名称（参考） | 作用 |
|---------|-----------------|------|
| **系统工具 → 模板配置** | `autoPkg` | 维护代码生成用的 **Package / 模板**（`package` + `template`），生成前需有可用模板包。 |
| **系统工具 → 代码生成器** | `autoCode` | **核心页面**：配置结构体/字段，勾选自动化选项，**预览代码**或**生成代码**。 |
| **系统工具 → 自动化代码管理** | `autoCodeAdmin` | 查看生成历史、回滚等（与生成器配合使用）。 |
| **超级管理 → 菜单管理** | — | 手动增删改菜单、组件路径、路由 `name` 等。 |
| **超级管理 → API 管理** | `api` | **同步 API**：把路由扫描进 `sys_apis`，供 Casbin 与前端鉴权使用。 |
| **超级管理 → 角色管理** | — | 给角色分配**菜单**与**API**，否则新菜单不可见或无权限。 |

## 二、推荐操作流程（从表到可点菜单）

### 1. 准备数据表

- 在业务库中建好表（字段、主键、注释等尽量完整，便于生成中文名与列表展示）。
- 若使用**多库**，需先在 **「系统工具」或配置中提供的 db-list** 里配置业务库别名，**重启后端**后再在生成器里选择「业务库」。

### 2.（可选）检查「模板配置」

路径：**系统工具 → 模板配置**

- 确认已有可用的 **package + 模板**（如 `package` 对应默认模板等）。
- 代码生成器里「Package」下拉选项来自此处维护的数据。

### 3. 打开「代码生成器」

路径：**系统工具 → 代码生成器**

页面主要分三块（自上而下）：

1. **AI 创建**（可选）  
   - 配置 `config.yaml` 中 `autocode` → `ai-path`（插件市场 AI-Path）后可使用「识图 / 生成」等能力。  
2. **从数据库创建**（常用）  
   - 选择 **业务库** → **数据库名** → **表名** → 点击 **「使用此表」**，自动拉取字段填充下方「字段列表」。  
3. **自动化结构**  
   - 填写 **结构名称、Package、简称、中文描述、表名** 等（可从表名自动带出部分字段）。  
   - 在表格中调整字段：**列表 / 表单 / 搜索 / 主键** 等勾选。

### 4. 基础设置与「一键」相关选项

在「基础设置 / 自动化设置」中，与「少动手」最相关的是：

| 选项 | 含义 |
|------|------|
| **生成前端** | 生成 Vue 页面、API 调用等。 |
| **生成后端** | 生成 Go 的 model / api / service / router 等（界面中可能部分固定勾选，以实际页面为准）。 |
| **自动创建 API** | 将生成的接口写入 **API 表**，便于后续 **API 同步** 与 Casbin 授权。 |
| **自动创建菜单** | 将菜单数据写入 **菜单表**，便于侧边栏挂载（仍需角色分配菜单）。 |
| **同步表结构** | 是否根据结构做 **AutoMigrate**（按需谨慎开启）。 |
| **创建按钮权限** | 生成页面内按钮级权限数据（需在角色里分配按钮）。 |
| **使用 GVA 结构** | 是否嵌入框架默认 `GVA_MODEL`（含 ID、时间戳等）。 |

默认表单中常见组合：**自动创建 API、自动创建菜单、同步表结构** 等为开启状态（以你当前项目 `web/src/view/systemTools/autoCode/index.vue` 默认值为准）。

### 5. 预览与生成

1. 点击 **「预览代码」**：不落地文件，仅查看将生成的内容。  
2. 点击 **「生成代码」**：调用后端 **`createTemp`** 接口，在配置的 `autocode` 根目录下写入 **server / web** 等文件（具体路径由 `config.yaml` 的 `autocode` 段 `root`、`web`、`server` 决定）。

生成成功后会提示类似：**「自动化代码创建成功，自动移动成功」**。

### 6. 注册业务路由（重要）

生成的是「模块代码」，**业务 Router 仍要在总路由里挂载**，否则接口 404。

- 常见位置：`server/initialize/router_biz.go`  
- 在 `initBizRouter` 中 `new(xxxRouter.RouterGroup).InitXxxRouter(privateGroup, publicGroup)` 一类注册，与现有 `content` 等模块写法保持一致。  
- 若生成器或文档提示需修改 `router/enter.go`、`api/v1/enter.go` 等 **enter 聚合文件**，按提示补全 `import` 与变量。

改完后 **重新编译并启动后端**：在 `server` 目录执行 `go run .` 或 `go build` 再运行。

### 7. 同步 API（超级管理 → API 管理）

路径：**超级管理 → API 管理**

1. 点击 **「同步 API」**。  
2. 在弹窗中按分组同步路由到数据库；不需要鉴权的路由可标记忽略。  
3. 同步后，新接口才会稳定参与 **Casbin 策略** 与前端按钮/API 校验。

### 8. 分配菜单与权限（角色管理）

路径：**超级管理 → 角色管理**

1. 编辑对应角色（如管理员/自定义角色）。  
2. 勾选**新菜单**（若已用「自动创建菜单」，一般会出现在菜单树中，也可能需微调父级、路径、`component`）。  
3. 勾选相关 **API**（与第 7 步同步结果一致）。  
4. 若使用了「创建按钮权限」，还需分配**按钮权限**。

保存后建议 **重新登录** 或刷新动态路由，使侧边栏出现新菜单并可正常访问 CRUD 页。

## 三、与「菜单点击」的关系（澄清）

- **不会**在「订单管理」「用户管理」这类业务菜单上凭空多出一个「一键生成」按钮。  
- **会**在 **系统工具 → 代码生成器** 中完成配置后，通过 **「自动创建菜单」+ 角色授权**，让新 CRUD 出现在左侧菜单，之后**日常就是普通菜单点击进入**。  
- 若未勾选自动创建菜单，则需在 **菜单管理** 里手动新增菜单项，并填写 `path`、`name`、`component` 等，再走角色授权。

## 四、配置项参考（`config.yaml`）

与生成路径、AI 相关的常见配置在 **`autocode`** 段，例如：

- `root`：生成代码相对/绝对根路径  
- `web`：前端子路径  
- `server`：后端子路径  
- `module`：Go module 路径片段  
- `ai-path`：AI 生成能力（可选）

修改后需 **重启后端** 生效。

## 五、注意事项

1. **开发环境使用为主**：页面警告条说明不建议未经审查直接用于生产。  
2. **生成前备份 / 使用 Git**：避免覆盖手写代码；必要时用 **自动化代码管理** 做回滚。  
3. **表名、结构名、简称、package** 等命名冲突时，生成器会校验并报错（如重复结构、package 与简称同名等）。  
4. 多数据库、插件包（`plugin` 模板）与单体 `package` 模板差异较大，以你选的模板与官方文档为准。

## 1. 总体架构（基于 gin-vue-admin）

- **后端**：Gin + GORM + JWT + Casbin（沿用现有体系）
- **前端**：Vue3 + Element-Plus（沿用现有体系，动态菜单）
- **路由分组**
  - `PrivateGroup`：需要登录与权限校验（文章 CRUD）
  - `PublicGroup`：公开访问（按 slug 获取已发布文章）

## 2. 模块分层落地（本仓库实现）

### 2.1 后端代码位置

- `server/model/content/content_article.go`

- `web/src/api/contentArticle.js`

## 3. 数据表设计

### 3.1 `content_articles`（文章表）

| 字段 | 类型 | 约束/索引 | 说明 |
| --- | --- | --- | --- |
| id | uint | PK | 主键 |
| created_at / updated_at / deleted_at | time | deleted_at index | 软删除 |
| author_id | uint | index | 作者用户 ID（sys_users.id） |
| title | varchar(200) | index | 标题 |
| slug | varchar(200) | uniqueIndex | SEO 友好链接标识 |
| summary | varchar(500) |  | 摘要 |
| content | longtext |  | 正文（MVP 先存 Markdown 文本） |
| content_type | varchar(20) |  | markdown/html |
| cover_image | varchar(500) |  | 封面图 |
| seo_title | varchar(200) |  | SEO 标题 |
| seo_keywords | varchar(500) |  | SEO 关键词（逗号分隔） |
| seo_description | varchar(500) |  | SEO 描述 |
| status | varchar(20) | index | draft/published/archived |
| published_at | time | index | 发布时间 |
| view_count | uint |  | 浏览量 |
| lead_count | uint |  | 线索量（后续） |

## 4. API 设计（MVP）

### 4.1 私有接口（需要 JWT + Casbin）

- `POST /api/contentArticle/createArticle`
- `PUT /api/contentArticle/updateArticle`
- `DELETE /api/contentArticle/deleteArticle`
- `GET /api/contentArticle/findArticle?id=1`
- `GET /api/contentArticle/getArticleList?page=1&pageSize=10&keyword=&status=`
- `POST /api/contentArticle/publishArticle`

### 4.2 公开接口（无需登录）

- `GET /api/public/article/:slug`

## 5. SEO 技术路线（第二期建议）

- **HTML 页面**：将公开访问从 JSON 扩展为 HTML（SSR/静态化/缓存）
- **站点地图**：`/sitemap.xml`，按 published 文章生成
- **robots**：`/robots.txt`
- **规范化链接**：canonical
